# my-sound-game
Это история и документация создания моей игры

Я начал с создания репозитория\
Создал пустой репозиторий на гит хаб, содержащий README.md\
На хосте же я написал в папке с уже сущствующи проектом\
git init\
git remote add origin https://github.com/egorzolotarev/my-sound-game\
Затем я сделал коммит, и запушил, все это с помощью IDE, и тут возникла проблема, ведь я хотел чтобы у меня весь код был в main ветке, а у меня создалась новая ветка master\
Чтобы их объединить, я использовал команды\
git branch -m master main - переименовал master в main\
git fetch origin - скачал содержимое с гита\
git merge origin/main --allow-unrelated-histories - склеил две независимые истории (тут создался коммит, который добавлял файлы из внешнего репозитория в локальный)\
git push -u origin main - запушил изменения в гит, при этом -u устанавливает связь между локальной и удаленной веткой, чтобы потом пушить без необходимости явно указывать main\

#Создание сетки(Grid)\
Изначально я захотел сделать поле, на которое впоследсивии будут ставиться блоки\
Для этого я создал Node2D Grid, и в скрипте в функции _Ready прописал создание линий, которые высчитывались относительно созданного мной свойства - количества линий по горизонтали и по вертикали\
Изначально я созал ее как Control? потому тчо унего есть Size, и чтобы у этого Control был размер, я сделал его ребенком CanvasLayer, однако впоследствии возникли проблемы с камерой, так как CanvasLayer не отслеживал изменения камеры и соответственно сетка стояла бы на месте как UI(лементы управления), поэтому я просто сделал ее Node2D, а размер взял от GetViewportRect().Size\
\
При этом для линий я создал отдельную сцену, чтобы можно было ее обобщить от Grid, и достаю я ее с помощью\
PackedScene lineForGridScene = ResourceLoader.Load<PackedScene>("res://line_for_grid.tscn");\
Line2D line = lineForGridScene.Instantiate<Line2D>();\

#Создание камеры(Camera2D)
Камеру я поместил в центре проекта, Grid же оставил в левом верхнем углу относительно нее\
При этом я настроил ее мастштаб и передвижение, были проблемы с передвижением, пока я не осознал что если Grid будет ее ребенком, он будет перемещаться вместе с ней и визуально движения вино не будет, хотя координаты будут менятся, поэтому я сделал их как 2 независимых узла\
Так же я сделал так чтоыб камера зависела от курсора мыши, это достигается тем что смотрятся координаты до, координаты после и камера смещается на их разницу.
Тут важно учитывать, что обрабатывать событие нужно только тогад когда колесико мыши зажато, иначе приближение сработает\
В дополнение ко всему я огранилил область движения камеры с помощью Limit в инспекторе, координаты по которым ее надо ограничить я подсмотрел у Grid, который находится в ее лувом верхнем углу
